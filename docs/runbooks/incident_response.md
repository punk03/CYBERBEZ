# Runbook: Реагирование на инциденты

## Обзор

Процедуры реагирования на инциденты безопасности в системе PROKVANT.

## Классификация инцидентов

### Критический (Critical)
- Активная атака на критическую инфраструктуру
- Ransomware атака
- Массовая утечка данных

### Высокий (High)
- Обнаружена APT атака
- Компрометация системы
- DDoS атака

### Средний (Medium)
- Обнаружена подозрительная активность
- Попытка несанкционированного доступа

### Низкий (Low)
- Ложные срабатывания
- Незначительные аномалии

## Процедуры реагирования

### Шаг 1: Обнаружение

1. Система автоматически обнаруживает угрозу
2. Отправляются алерты операторам
3. Угроза отображается в dashboard

### Шаг 2: Оценка

1. Просмотреть детали угрозы
2. Оценить критичность
3. Определить затронутые системы

### Шаг 3: Изоляция

1. Система автоматически применяет изоляцию (если настроено)
2. Или оператор подтверждает изоляцию вручную
3. Проверить статус изоляции

### Шаг 4: Анализ

1. Собрать логи и метрики
2. Проанализировать атаку
3. Определить root cause

### Шаг 5: Восстановление

1. Устранить угрозу
2. Восстановить затронутые системы
3. Проверить работоспособность

### Шаг 6: Документирование

1. Задокументировать инцидент
2. Обновить правила детекции
3. Провести post-mortem анализ

## Автоматические действия

Система автоматически:
- Изолирует источник угрозы
- Блокирует трафик
- Переключает на резервные системы (для критических угроз)
- Отправляет уведомления

## Ручные действия

Оператор должен:
- Подтвердить критические действия
- Проанализировать инцидент
- Документировать результаты

## Эскалация

Если инцидент не может быть решен автоматически:
1. Уведомить старшего оператора
2. Уведомить менеджера инцидентов
3. При необходимости - внешних специалистов
